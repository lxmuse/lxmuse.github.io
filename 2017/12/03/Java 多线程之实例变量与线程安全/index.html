而且上述结果，看起来非常奇怪，与期望的相差甚远！这就是线程安全问题！

以上述例子为例，某些 JVM 中，i– 实际是三步操作：
1）获取原 i 值
2）计算 i-1
3) 对 i 进行赋值

正是因为分步操作，可能在执行完第一步以后，线程执行就会切换，导致执行结果千奇百怪，和期望不符。

若想解决线程安全问题，则需要执行线程同步策略。

synchronized 关键字修改线程内的方法，可以使其即使被多个线程调用，其也是按顺序执行，从而保证线程安全。
